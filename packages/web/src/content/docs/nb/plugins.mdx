## Bruk av en plugin

### Fra lokale filer

### Hvordan plugins installeres

**npm-plugins** installeres automatisk ved hjelp av Bun ved oppstart. Pakker og deres avhengigheter er bufret i `~/.cache/voracode/node_modules/`.

**Lokale plugins** lastes direkte fra plugin-katalogen. For 횄짜 bruke eksterne pakker, m횄짜 du opprette en `package.json` i konfigurasjonskatalogen din (se [Dependencies](#dependencies)), eller publisere plugin-en til npm og [legg den til i konfigurasjonen din](/docs/config#plugins).

---

### Lasterekkef횄쨍lge

Plugins lastes inn fra alle kilder og alle kroker kj횄쨍res i rekkef횄쨍lge. Lastrekkef횄쨍lgen er:

1. Global konfigurasjon (`~/.config/voracode/voracode.json`)
2. Prosjektkonfigurasjon (`voracode.json`)
3. Global plugin-katalog (`~/.config/voracode/plugins/`)
4. Prosjektpluginkatalog (`.voracode/plugins/`)

Dupliserte npm-pakker med samme navn og versjon lastes inn 횄짤n gang. Imidlertid lastes en lokal plugin og en npm plugin med lignende navn begge separat.

## Opprette en plugin

En plugin er en **JavaScript/TypeScript-modul** som eksporterer en eller flere plugin-moduler
funksjoner. Hver funksjon mottar et kontekstobjekt og returnerer et krokobjekt.

---

### Avhengigheter

Lokale plugins og tilpassede verkt횄쨍y kan bruke eksterne npm-pakker. Legg til en `package.json` til konfigurasjonskatalogen med avhengighetene du trenger.

```json title=".voracode/package.json"
{
  "dependencies": {
    "shescape": "^2.1.0"
  }
}
```

Voracode kj횄쨍rer `bun install` ved oppstart for 횄짜 installere disse. Programtilleggene og verkt횄쨍yene dine kan deretter importere dem.

```ts title=".voracode/plugins/my-plugin.ts"
import { escape } from "shescape"

export const MyPlugin = async (ctx) => {
  return {
    "tool.execute.before": async (input, output) => {
      if (input.tool === "bash") {
        output.args.command = escape(output.args.command)
      }
    },
  }
}
```

---

### Grunnleggende struktur

```js title=".voracode/plugins/example.js"
export const MyPlugin = async ({ project, client, $, directory, worktree }) => {
  console.log("Plugin initialized!")

  return {
    // Hook implementations go here
  }
}
```

Plugin-funksjonen mottar:

- `project`: Gjeldende prosjektinformasjon.
- `directory`: Gjeldende arbeidskatalog.
- `worktree`: Git-arbeidstrebanen.
- `client`: En Voracode SDK klient for samhandling med AI.
- `$`: Buns [shell API](https://bun.com/docs/runtime/shell) for 횄짜 utf횄쨍re kommandoer.

### Send varsler

}

### Egendefinerte verkt횄쨍y

Plugins kan ogs횄짜 legge til egendefinerte verkt횄쨍y til Voracode:

```ts title=".voracode/plugins/custom-tools.ts"
import { type Plugin, tool } from "@voracode-ai/plugin"

export const CustomToolsPlugin: Plugin = async (ctx) => {
  return {
    tool: {
      mytool: tool({
        description: "This is a custom tool",
        args: {
          foo: tool.schema.string(),
        },
        async execute(args, context) {
          const { directory, worktree } = context
          return `Hello ${args.foo} from ${directory} (worktree: ${worktree})`
        },
      }),
    },
  }
}
```

`tool`-hjelperen lager et tilpasset verkt횄쨍y som Voracode kan kalle. Den tar en Zod-skjemafunksjon og returnerer en verkt횄쨍ydefinisjon med:

- `description`: Hva verkt횄쨍yet gj횄쨍r
- `args`: Zod-skjema for verkt횄쨍yets argumenter
  - `execute`: Funksjon som kj횄쨍rer n횄짜r verkt횄쨍yet kalles

Dine egendefinerte verkt횄쨍y vil v횄짝re tilgjengelige for Voracode sammen med innebygde verkt횄쨍y.

:::note
Hvis et plugin-verkt횄쨍y bruker samme navn som et innebygd verkt횄쨍y, vil plugin-verkt횄쨍yet ha forrang.
:::

### Komprimerings-hooks

Tilpass konteksten inkludert n횄짜r en 횄쨍kt komprimeres:

```ts title=".voracode/plugins/compaction.ts"
import type { Plugin } from "@voracode-ai/plugin"

export const CompactionPlugin: Plugin = async (ctx) => {
  return {
    "experimental.session.compacting": async (input, output) => {
      // Inject additional context into the compaction prompt
      output.context.push(`
## Custom Context

Include any state that should persist across compaction:
- Current task status
- Important decisions made
- Files being actively worked on
`)
    },
  }
}
```

`experimental.session.compacting`-kroken trigges f횄쨍r LLM genererer et fortsettelsessammendrag. Bruk den til 횄짜 injisere domenespesifikk kontekst som standard komprimeringsprompt ville g횄짜 glipp av.

Du kan ogs횄짜 erstatte komprimeringsprompten helt ved 횄짜 stille inn `output.prompt`:

```ts title=".voracode/plugins/custom-compaction.ts"
import type { Plugin } from "@voracode-ai/plugin"

export const CustomCompactionPlugin: Plugin = async (ctx) => {
  return {
    "experimental.session.compacting": async (input, output) => {
      // Replace the entire compaction prompt
      output.prompt = `
You are generating a continuation prompt for a multi-agent swarm session.

Summarize:
1. The current task and its status
2. Which files are being modified and by whom
3. Any blockers or dependencies between agents
4. The next steps to complete the work

Format as a structured prompt that a new agent can use to resume work.
`
    },
  }
}
```

N횄짜r `output.prompt` er angitt, erstatter den standard komprimeringsprompt fullstendig. `output.context`-matrisen ignoreres i dette tilfellet.
