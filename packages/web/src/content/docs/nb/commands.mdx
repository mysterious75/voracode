## Lag kommandofiler

Lag nedmerkingsfiler i `commands/`-katalogen for 횄짜 definere egendefinerte kommandoer.

Opprett `.voracode/commands/test.md`:

```md title=".voracode/commands/test.md"
---
description: Run tests with coverage
agent: build
model: anthropic/claude-3-5-sonnet-20241022
---

Run the full test suite with coverage report and show any failures.
Focus on the failing tests and suggest fixes.
```

Frontmatter definerer kommandoegenskaper. Innholdet blir malen.

Bruk kommandoen ved 횄짜 skrive `/` etterfulgt av kommandonavnet.

```bash frame="none"
"/test"
```

## Konfigurasjon

Du kan legge til egendefinerte kommandoer gjennom Voracode-konfigurasjonen eller ved 횄짜 lage markdown-filer i katalogen `commands/`.

---

### JSON

Bruk alternativet `command` i Voracode [config](/docs/config):

```json title="voracode.jsonc" {4-12}
{
  "$schema": "https://voracode.ai/config.json",
  "command": {
    // This becomes the name of the command
    "test": {
      // This is the prompt that will be sent to the LLM
      "template": "Run the full test suite with coverage report and show any failures.\nFocus on the failing tests and suggest fixes.",
      // This is shown as the description in the TUI
      "description": "Run tests with coverage",
      "agent": "build",
      "model": "anthropic/claude-3-5-sonnet-20241022"
    }
  }
}
```

N횄짜 kan du kj횄쨍re denne kommandoen i TUI:

```bash frame="none"
/test
```

---

### Markdown

Du kan ogs횄짜 definere kommandoer ved 횄짜 bruke markdown-filer. Plasser dem i:

- Globalt: `~/.config/voracode/commands/`
- Per prosjekt: `.voracode/commands/`

```markdown title="~/.config/voracode/commands/test.md"
---
description: Run tests with coverage
agent: build
model: anthropic/claude-3-5-sonnet-20241022
---

Run the full test suite with coverage report and show any failures.
Focus on the failing tests and suggest fixes.
```

Markdown-filnavnet blir kommandonavnet. For eksempel lar `test.md`
deg kj횄쨍re:

```bash frame="none"
/test
```

## Ledetekstkonfigurasjon

---

### Argumenter

Send argumenter til kommandoer ved 횄짜 bruke plassholderen `$ARGUMENTS`.

```md title=".voracode/commands/component.md"
---
description: Create a new component
---

Create a new React component named $ARGUMENTS with TypeScript support.
Include proper typing and basic structure.
```

Kj횄쨍r kommandoen med argumenter:

```bash frame="none"
/component Button
```

Og `$ARGUMENTS` vil bli erstattet med `Button`.

Du kan ogs횄짜 f횄짜 tilgang til individuelle argumenter ved 횄짜 bruke posisjonelle parametere:

- `$1` - F횄쨍rste argument
- `$2` - Andre argument
- `$3` - Tredje argument
- Og s횄짜 videre...

For eksempel:

```md title=".voracode/commands/create-file.md"
---
description: Create a new file with content
---

Create a file named $1 in the directory $2
with the following content: $3
```

Kj횄쨍r kommandoen:

```bash frame="none"
/create-file config.json src "{ \"key\": \"value\" }"
```

Dette erstatter:

- `$1` med `config.json`
- `$2` med `src`
- `$3` med `{ "key": "value" }`

### Shell-utdata

Bruk _!`command`_ for 횄짜 injisere [bash-kommando](/docs/tui#bash-commands)-utdata i ledeteksten.

For eksempel, for 횄짜 lage en egendefinert kommando som analyserer testdekning:

```md title=".voracode/commands/analyze-coverage.md"
---
description: Analyze test coverage
---

Here are the current test results:
!`npm test`

Based on these results, suggest improvements to increase coverage.
```

Eller for 횄짜 se gjennom nylige endringer:

```md title=".voracode/commands/review-changes.md"
---
description: Review recent changes
---

Recent git commits:
!`git log --oneline -10`

Review these changes and suggest any improvements.
```

Kommandoer kj횄쨍res i prosjektets rotkatalog og utdataene deres blir en del av ledeteksten.

---

### Filreferanser

Inkluder filer i kommandoen din ved 횄짜 bruke `@` etterfulgt av filnavnet.

```md title=".voracode/commands/review-component.md"
---
description: Review component
---

Review the component in @src/components/Button.tsx.
Check for performance issues and suggest improvements.
```

Filinnholdet blir automatisk inkludert i ledeteksten.

## Alternativer

La oss se p횄짜 konfigurasjonsalternativene i detalj.

---

### Mal

Alternativet `template` definerer ledeteksten som vil bli sendt til LLM n횄짜r kommandoen utf횄쨍res.

```json title="voracode.json"
{
  "command": {

    }
  }

```

Dette er et **obligatorisk** konfigurasjonsalternativ.

---

### Beskrivelse

```json title="voracode.json"
    }
  }
}
```

Dette vises som beskrivelsen i TUI n횄짜r du skriver inn kommandoen.

---

### Agent

Bruk `agent`-konfigurasjonen for 횄짜 spesifisere hvilken [agent](/docs/agents) som skal utf횄쨍re denne kommandoen.
Hvis dette er en [subagent](/docs/agents/#subagents), vil kommandoen utl횄쨍se en subagent p횄짜kalling som standard.
For 횄짜 deaktivere denne virkem횄짜ten, sett `subtask` til `false`.

```json title="voracode.json"
{
  "command": {
    "review": {
      "agent": "plan"
    }
  }
}
```

Denne kommandoen vil veilede deg gjennom 횄짜 lage en ny agent med en tilpasset systemmelding og verkt횄쨍ykonfigurasjon.

---

### Underoppgave

Bruk `subtask` for 횄짜 tvinge kommandoen til 횄짜 utl횄쨍se en [subagent](/docs/agents/#subagents) p횄짜kalling.
Dette er nyttig hvis du vil at kommandoen ikke skal forurense din prim횄짝re kontekst og vil **tvinge** agenten til 횄짜 fungere som en underagent,
selv om `mode` er satt til `primary` p횄짜 [agent](/docs/agents) konfigurasjonen.

```json title="voracode.json"
{
  "command": {
    "analyze": {
      "subtask": true
    }
  }
}
```

Dette er et **valgfritt** konfigurasjonsalternativ.

---

### Modell

Bruk `model`-konfigurasjonen for 횄짜 overstyre standardmodellen for denne kommandoen.

```json title="voracode.json"

    "analyze": {

    }
  }
}
```

Dette er et **valgfritt** konfigurasjonsalternativ.

## Innebygd

Voracode inkluderer flere innebygde kommandoer som `/init`, `/undo`, `/redo`, `/share`, `/help`; [finn ut mer](/docs/tui#commands).

:::note
Egendefinerte kommandoer kan overstyre innebygde kommandoer.
:::

Hvis du definerer en egendefinert kommando med samme navn, vil den overstyre den innebygde kommandoen.
