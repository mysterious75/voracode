While Voracode can run directly on Windows, we recommend using [Windows Subsystem for Linux (WSL)](https://learn.microsoft.com/en-us/windows/wsl/install) for the best experience. WSL provides a Linux environment that works seamlessly with Voracode's features.

:::tip[Why WSL?]
WSL offers better file system performance, full terminal support, and compatibility with development tools that Voracode relies on.
:::

---

## Setup

<Steps>

1. **Install WSL**

   If you haven't already, [install WSL](https://learn.microsoft.com/en-us/windows/wsl/install) using the official Microsoft guide.

2. **Install Voracode in WSL**

   Once WSL is set up, open your WSL terminal and install Voracode using one of the [installation methods](/docs/).

   ```bash
   curl -fsSL https://voracode.ai/install | bash
   ```

3. **Use Voracode from WSL**

   Navigate to your project directory (access Windows files via `/mnt/c/`, `/mnt/d/`, etc.) and run Voracode.

   ```bash
   cd /mnt/c/Users/YourName/project
   voracode
   ```

</Steps>

## Desktop App + WSL Server

If you prefer using the Voracode Desktop app but want to run the server in WSL:

1. **Start the server in WSL** with `--hostname 0.0.0.0` to allow external connections:

   ```bash
   voracode serve --hostname 0.0.0.0 --port 4096
   ```

2. **Connect the Desktop app** to `http://localhost:4096`

:::note
If `localhost` does not work in your setup, connect using the WSL IP address instead (from WSL: `hostname -I`) and use `http://<wsl-ip>:4096`.
:::

:::caution
When using `--hostname 0.0.0.0`, set `VORACODE_SERVER_PASSWORD` to secure the server.
:::

```bash
VORACODE_SERVER_PASSWORD=your-password voracode serve --hostname 0.0.0.0
```

## Web Client + WSL

For the best web experience on Windows:

1. **Run `voracode web` in the WSL terminal** rather than PowerShell:

   ```bash
   voracode web --hostname 0.0.0.0
   ```

2. **Access from your Windows browser** at `http://localhost:<port>` (Voracode prints the URL)

Running `voracode web` from WSL ensures proper file system access and terminal integration while still being accessible from your Windows browser.

## Tips

- Keep Voracode running in WSL for projects stored on Windows drives - file access is seamless
- Use VS Code's [WSL extension](https://code.visualstudio.com/docs/remote/wsl) alongside Voracode for an integrated development workflow
- Your Voracode config and sessions are stored within the WSL environment at `~/.local/share/voracode/`
