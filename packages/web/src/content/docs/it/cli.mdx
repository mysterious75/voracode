La CLI di Voracode, per impostazione predefinita, avvia la [TUI](/docs/tui) quando viene eseguita senza argomenti.

```bash
voracode
```

Ma accetta anche i comandi documentati in questa pagina. Questo ti permette di interagire con Voracode in modo programmatico.

```bash
voracode run "Explain how closures work in JavaScript"
```

---

### tui

Avvia l'interfaccia testuale di Voracode (interfaccia utente da terminale).

```bash
voracode [project]
```

#### Flag

| Flag                                     | Breve | Descrizione                                                              |
| ---------------------------------------- | ----- | ------------------------------------------------------------------------ |
| <nobr><code>{"--continue"}</code></nobr> | `-c`  | Continua l'ultima sessione                                               |
| <nobr><code>{"--session"}</code></nobr>  | `-s`  | ID sessione da continuare                                                |
| <nobr><code>{"--fork"}</code></nobr>     |       | Duplica la sessione quando continui (usa con `--continue` o `--session`) |
| <nobr><code>{"--prompt"}</code></nobr>   |       | Prompt da usare                                                          |
| <nobr><code>{"--model"}</code></nobr>    | `-m`  | Modello nel formato provider/model                                       |
| <nobr><code>{"--agent"}</code></nobr>    |       | Agente da usare                                                          |
| <nobr><code>{"--port"}</code></nobr>     |       | Porta su cui mettersi in ascolto                                         |

### attach

Collega un terminale a un backend server di Voracode giÃ  in esecuzione avviato tramite i comandi `serve` o `web`.

```bash
voracode attach [url]
```

Questo consente di usare la TUI con un backend Voracode remoto. Per esempio:

```bash
# Start the backend server for web/mobile access
voracode web --port 4096 --hostname 0.0.0.0

# In un altro terminale, collega la TUI al backend in esecuzione
voracode attach http://10.20.30.40:4096
```

#### Flag

| Flag                                     | Breve | Descrizione                                                                                     |
| ---------------------------------------- | ----- | ----------------------------------------------------------------------------------------------- |
| <nobr><code>{"--dir"}</code></nobr>      |       | Working directory in cui avviare la TUI                                                         |
| <nobr><code>{"--continue"}</code></nobr> | `-c`  | Continua l'ultima sessione                                                                      |
| <nobr><code>{"--session"}</code></nobr>  | `-s`  | ID sessione da continuare                                                                       |
| <nobr><code>{"--fork"}</code></nobr>     |       | Crea un fork della sessione durante la continuazione (usa con `--continue` o `--session`)       |
| <nobr><code>{"--password"}</code></nobr> | `-p`  | Password per l'autenticazione di base (predefinita: `VORACODE_SERVER_PASSWORD`)                 |
| <nobr><code>{"--username"}</code></nobr> | `-u`  | Nome utente per l'autenticazione di base (predefinito: `VORACODE_SERVER_USERNAME` o `voracode`) |

### auth

Comando per gestire le credenziali e il login dei provider.

```bash
voracode auth [command]
```

---

#### login

Voracode si basa sull'elenco provider di [Models.dev](https://models.dev), quindi puoi usare `voracode auth login` per configurare le API key per qualunque provider tu voglia usare. Le credenziali vengono salvate in `~/.local/share/voracode/auth.json`.

```bash
voracode auth login
```

Quando Voracode si avvia, carica i provider dal file delle credenziali e, se presenti, anche eventuali key definite nell'ambiente o in un file `.env` nel progetto.

---

#### list

Elenca tutti i provider autenticati come salvati nel file delle credenziali.

```bash
voracode auth list
```

Oppure la versione corta.

```bash
voracode auth ls
```

---

#### logout

Esegue il logout da un provider rimuovendolo dal file delle credenziali.

```bash
voracode auth logout
```
