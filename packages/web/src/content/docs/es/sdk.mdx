## Salida Estructurada

Puede solicitar una salida JSON estructurada del modelo especificando un `format` con un esquema JSON. El modelo utilizarÃ¡ una herramienta `StructuredOutput` para devolver un JSON validado que coincida con su esquema.

### Uso BÃ¡sico

```typescript

  path: { id: sessionId },
  body: {

      type: "json_schema",
      schema: {
        type: "object",
        properties: {
          company: { type: "string", description: "Company name" },
          founded: { type: "number", description: "Year founded" },
          products: {
            type: "array",
            items: { type: "string" },
            description: "Main products",
          },
        },
        required: ["company", "founded"],
      },
    },
  },
})

// Access the structured output
console.log(result.data.info.structured_output)
// { company: "Anthropic", founded: 2021, products: ["Claude", "Claude API"] }
```

### Tipos de Formato de Salida

| Tipo          | DescripciÃ³n                                                           |
| ------------- | --------------------------------------------------------------------- |
| `text`        | Predeterminado. Respuesta de texto estÃ¡ndar (sin salida estructurada) |

### Formato de Esquema JSON

Cuando use `type: 'json_schema'`, proporcione:

| Campo        | Tipo            | DescripciÃ³n                                                      |
| ------------ | --------------- | ---------------------------------------------------------------- |
| `type`       | `'json_schema'` | Requerido. Especifica el modo de esquema JSON                    |
| `schema`     | `object`        | Requerido. Objeto JSON Schema que define la estructura de salida |
| `retryCount` | `number`        | Opcional. NÃºmero de reintentos de validaciÃ³n (predeterminado: 2) |

### Manejo de Errores

Si el modelo no logra producir una salida estructurada vÃ¡lida despuÃ©s de todos los reintentos, la respuesta incluirÃ¡ un `StructuredOutputError`:

```typescript

  console.error("Failed to produce structured output:", result.data.info.error.message)
```

### Mejores PrÃ¡cticas

1. **Proporcione descripciones claras** en las propiedades de su esquema para ayudar al modelo a entender quÃ© datos extraer
2. **Use `required`** para especificar quÃ© campos deben estar presentes

4. **Establezca un `retryCount` apropiado** - aumente para esquemas complejos, disminuya para simples
